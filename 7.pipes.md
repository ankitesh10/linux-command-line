# Pipes

## Pipe Command Output

```sh
cat ls.txt | grep "ls-error.txt"
```

- The output of `cat` is piped into `grep`.
- `grep` takes text and finds matching text.

## Find a Process

```sh
ps aux | grep "ps aux"
```

## Run a Background Process

```sh
yes > /dev/null &
ps aux | grep "yes"
kill -9 209
```

- `yes > /dev/null &` runs `yes` in the background and logs to `/dev/null`.
- `ps aux | grep "yes"` finds the running process.
- `kill -9 209` kills the process with PID `209`.

## Pipe `yes` into an Interactive Command

```sh
yes y | rm -i file*
```

## Note

This command returns two results:

```sh
ps aux | grep "ps aux"
```

Example output:

```text
root       251  0.0  0.0   7632  3632 pts/0    R+   15:38   0:00 ps aux
root       252  0.0  0.0   3128  1512 pts/0    S+   15:38   0:00 grep --color=auto ps aux
```

The second result is the `grep` command itself.
