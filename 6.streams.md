# Streams

## Output Streams

```sh
command > out.txt
command 1> out.txt
command 2> err.txt
command > out.txt 2>&1
```

- `>` redirects `stdout` to a file.
- `1>` is the same as `>`, because `1` is `stdout`.
- `2>` redirects `stderr` to a file.
- `2>&1` redirects `stderr` to wherever `stdout` is going.

## Append Output

```sh
ls 1>> out.txt
```

`>>` appends output instead of replacing the file.

## Redirect Each Stream Separately

```sh
ls -lsah 1> stdout.txt 2> stderr.txt
```

This redirects `stdout` and `stderr` to different files.

## Input Streams

```sh
cat < file.txt
grep "out.txt" < out.txt
grep "out.txt" < out.txt 1> grep.txt 2> /dev/null
```

- `cat` technically reads files for you without requiring `<`.
- `grep "out.txt" < out.txt 1> grep.txt 2> /dev/null` redirects the result into `grep.txt` and ignores errors by sending them to `/dev/null`.
- `/dev/null` is like a void in Linux.
