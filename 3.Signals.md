# Signals

## Run a Background Task

```sh
yes > /dev/null &
```

1. `yes` runs repeatedly.
2. `>` is the shell output redirection operator. It takes the standard output (`stdout`) of the command on its left (`yes`) and redirects it somewhere else instead of printing to the terminal.
3. `/dev/null` is a special file in Unix-like systems often called a "bit bucket" or "black hole". Anything written to `/dev/null` is immediately discarded.
4. `&` is the shell background operator. Putting `&` at the end of a command tells the shell to run that command in the background.

That means:

- The `yes` process keeps running.
- Your shell prompt returns immediately and you can type new commands.
- The process is detached from the current foreground job. You manage it with commands like `ps`, `jobs`, and `kill`, not with `Ctrl+C` in the foreground.

## Check Active Processes

```sh
ps aux | grep yes
```

To kill the background task, send it a kill signal:

```sh
kill -9 <PID>
```

- To see what `9` means, run `kill -l`.
- `9` is `SIGKILL`.

## What is a Signal?

A signal is a notification that you send to a program. It's up to the program to understand what to do with that.

Programs are not obligated to respond to every signal, but most follow the expected behavior. If you send a program a message to quit, it usually quits.

## `CTRL + C` - `SIGINT`

If you hit `CTRL + C` while a program is running, you're telling it to interrupt what it's doing and stop.

You'll use this constantly. One reason is that some processes are designed to never quit until you send them a `SIGINT`.
A good example is the `yes` command. All `yes` does is spam the string `y` until you tell it to quit.

Try running:

```sh
yes
```

To stop it, hit `CTRL + C`.

Another good use case is when you started running a command and did not mean to. You can send a `SIGINT` and start again.

Some programs may not respect `CTRL + C`, and nearly all of them will take time to clean up after themselves when they receive a `SIGINT`.

## `CTRL + D` - `SIGQUIT`

`CTRL + D` is less useful, but still good to know.

Many programs won't respond to a `SIGQUIT`, but Bash itself will. If you're in a Bash prompt and want it to exit, hit `CTRL + D`.
You can also close the window or type:

```sh
exit
```

## `SIGTERM`

There is no shortcut for `SIGTERM`, but it is important to know it exists.

If you use the `kill` program to kill another program, it does that by sending a `SIGTERM` to the program.

## `SIGKILL`

If you want a program to stop immediately, use:

```sh
kill -9 <PID>
```

You can also write:

```sh
kill -SIGKILL <PID>
```

This tells the program: do not clean up, just stop as soon as possible.

## More Signals

Run this to see all the signals your computer supports:

```sh
kill -l
```

Most signals are used for processes to communicate with each other or with the shell. For now, the signals above are the ones you care about most.
